# 系统功能测试与验证

本章针对本 KV 存储系统的核心功能进行系统性测试，包括：

* 多引擎 KV 存储
* Key 过期机制
* 持久化（全量 + 增量）
* 主从复制
* 内存管理策略
* 批量命令处理

测试均基于 RESP 协议，通过 `redis-cli` 或 `nc` 进行交互验证。

---

# 1. 测试环境

## 1.1 运行环境

* 操作系统：Linux（Ubuntu）
* 编译工具：gcc / make
* 网络工具：redis-cli / nc

## 1.2 启动方式

```bash
./kvstore --port 5000 --mem libc
```

系统启动后将完成：

* 三种 KV 引擎初始化
* 过期管理初始化
* 持久化文件加载（dump + aof）
* 启动 reactor 网络事件循环

---

# 2. KV 存储功能测试

系统支持三种存储引擎：


| 引擎   | 命令前缀 | 特点         |
| ------ | -------- | ------------ |
| Array  | 无前缀   | 简单线性存储 |
| Hash   | H        | 哈希表       |
| RBTree | R        | 有序存储     |

---

## 2.1 基本 KV 操作测试（Array）

### 测试步骤

```bash
SET a1 v1
GET a1
MOD a1 v2
GET a1
DEL a1
GET a1
```

### 预期结果

* SET/MOD 返回 OK
* GET 返回正确值
* DEL 后 GET 返回空

### 测试结论

验证了数组引擎的：

* 插入
* 查询
* 修改
* 删除

---

## 2.2 Hash 引擎测试

### 测试步骤

```bash
HSET h1 v1
HGET h1
HMOD h1 v2
HGET h1
HDEL h1
```

### 预期结果

* 数据读写正常
* 修改生效
* 删除后不存在

### 测试结论

验证哈希表：

* 冲突处理（链表）
* 查询效率
* 动态更新

---

## 2.3 RBTree 引擎测试

### 测试步骤

```bash
RSET r1 v1
RSET r2 v2
RSET r3 v3
RGET r2
RDEL r2
```

### 预期结果

* 插入成功
* 查询准确
* 删除后不存在

### 测试结论

验证红黑树：

* 平衡性维护
* 查找性能
* 删除修复机制

---

# 3. Key 过期机制测试

系统采用：

* 惰性删除（访问触发）
* 主动删除（周期扫描）

---

## 3.1 基本 TTL 测试

### 测试步骤

```bash
SET t1 v1
EXPIRE t1 3
TTL t1
sleep 4
GET t1
```

### 预期结果

* TTL 初始为正数
* 过期后 GET 返回空

### 测试结论

验证：

* 过期时间记录正确
* key 自动删除

---

## 3.2 PERSIST 测试

```bash
SET p1 v1
EXPIRE p1 5
PERSIST p1
```

### 预期结果

* TTL 变为 -1
* 数据不会被删除

---

## 3.3 多引擎 TTL 独立性测试

```bash
SET k v1
HSET k v2
RSET k v3
EXPIRE k 2
HEXPIRE k 5
REXPIRE k 8
```

### 预期结果

不同引擎的 key 独立过期

# 9. 总体测试结论

通过上述测试可以验证：

1. 三种 KV 引擎功能完整且互不干扰

3. 持久化支持全量 + 增量恢复

5. 内存策略可切换且可观测

7. 网络层支持并发与非阻塞通信

系统整体功能完整，设计合理，具备类 Redis 的核心能力。
